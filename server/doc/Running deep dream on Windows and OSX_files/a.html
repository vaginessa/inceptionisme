                                                If you want to save the images to your local drive that are generated by that looping code, just redirect them to the /src/ drive you have mapped. I've added more options into the call to the deepdream function too, just mess with the numbers or change the end = bit for a different layer.

                                                e.g.

                                                !mkdir '/src/frames'
                                                frame = img
                                                frame_i = 0

                                                h, w = frame.shape[:2]
                                                s = 0.05 # scale coefficient
                                                for i in xrange(100):
                                                frame = deepdream(net, frame, iter_n=10, octave_n=4, octave_scale=1.4, end='inception_4c/output', clip=True)
                                                PIL.Image.fromarray(np.uint8(frame)).save("/src/frames/%04d.jpg"%frame_i)
                                                frame = nd.affine_transform(frame, [1-s,1-s,1], [h*s/2,w*s/2,0], order=1)
                                                frame_i += 1

                                                                                                          Olluf 8 days ago

                                                                                                          Just leaving these things here in case they help other Windows users:
                                                                                                          1) "TLS-enabled daemon without TLS"-error: You may have to turn on virtualization from BIOS. Restart computer, interrupt normal startup, go to BIOS, Security-tab, and you'll find it.
                                                                                                          2) "Kernel has died"-error: Run boot2docker delete, boot2docker init, start up Oracle VM VirtualBox (Go to start menu and type oracle, it'll come up), open boot2docker-vm Settings and adjust your CPU number in Processor tab, your Base Memory in Motherboard tab and your Video Memory in Video tab (all these should be within the green area of the sliders). Then run boot2docker up and carry on with the "docker run -d -p 433:..."-command.
                                                                                                          3) "valid for 127.0.0.1"-error: Run these commands: boot2docker delete, boot2docker init, boot2docker up. Then carry on with the "docker run -d -p 433:..."-command.
                                                                                                          4) Do not attempt to place your repo path in another drive, just put it in //c/Users/your-user-name/Pictures:/src as instructed to avoid hassle. Apparently this is also case-sensitive (capital U, capital P)
