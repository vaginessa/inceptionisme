Container Cluster Operations

  2. Creating a container cluster
  3. Viewing a container cluster
  4. Deleting a container cluster

Before you begin

Follow the directions to:

  • Enable the API.

  • Install the gcloud and kubectl command line interfaces.

Creating a container cluster

Container clusters can be created using the gcloud command line interface, or through the Developers Console.

A container cluster consists of the specified number and type of Google Compute Engine instances. Container Engine takes care of managing the Kubernetes API.

The cluster is created in the project's default network.

One firewall is added for the cluster. After cluster creation, the cluster creates routes for each node to allow the containers on that node to communicate with all other instances in the cluster.

Developers Console

You can create and delete container clusters in the Google Developers Console.

  1. Visit the Developers Console at https://console.developers.google.com/.
  2. Select your Container Engine project. If you haven't activated the Container Engine API, read and follow the directions in Before You Begin.
  3. In the left nav, select Compute then Container Engine.

  4. Click New container cluster. The following fields are required:

    • Name: The name of this container cluster. It must be unique within the project and the zone.
    • Zone: The Google Compute Engine zone in which to create the container cluster.
    • Cluster size: The number of instances to include in this container cluster. You must have enough quota for the number of nodes plus one (to include the master) as well as sufficient route and firewall quota.
    • Machine type: The Compute Engine machine type to use for the instances. Each machine type is billed differently. For pricing information, review the price sheet.

  5. Click Create to create the container cluster.

Gcloud command line interface

Container clusters are created using the container clusters sub-command of gcloud beta.

$ gcloud beta container clusters create NAME --zone ZONE

Optional flags:

  • --machine-type: The Google Compute Engine machine type (e.g. n1-standard-1) to use for instances in this container cluster. If unspecified, the default machine type is n1-standard-1.
  • --num-nodes: Specifies the number of nodes to create in the container cluster. Default is 3. Your Developers Console project must have enough quota for the number of nodes plus one (to include the master) as well as sufficient route and firewall quota.
  • --password: The password to use for authentication to the master. The service will create a strong password if this flag is not specified. If specifying your own password, make sure to create a strong one, as the master endpoint is open to the public Internet.
  • --username: The username to use for authentication to the master. Default is admin.

A successful container cluster create request returns the cluster resource:

Creating cluster doc-example...done.
Created [.../projects/container-engine-docs/zones/us-central1-a/clusters/doc-example].
kubeconfig entry generated for doc-example. To switch context to the cluster, run

$ kubectl config use-context gke_container-engine-docs_us-central1-a_doc-example

NAME         ZONE           CLUSTER_API_VERSION  MASTER_IP  MACHINE_TYPE   STATUS
doc-example  us-central1-a  0.19.1               x.x.x.x    n1-standard-1  running

Tip: The note about kubectl config use-context in the create response gives directions for switching between clusters when using kubectl. By creating the cluster, you are automatically setting the context to that cluster, so the command isn't required at this point.

Viewing a container cluster

Developers Console

To view your container clusters in the Developers Console:

  1. Visit the Developers Console.
  2. Select the appropriate project.
  3. In the left navigation menu, select Compute and then Container Engine.

Your container clusters are listed. Click a cluster name for more information about that cluster.

Gcloud command line interface

To view a particular container cluster with gcloud beta, use the clusters describe sub-command:

$ gcloud beta container clusters describe NAME --zone ZONE

To list the project's container clusters within a specific zone with gcloud beta, use the clusters list sub-command:

$ gcloud beta container clusters list --zone ZONE

Deleting a container cluster

Deleting a container cluster deletes all of the node instances, the master instance, any pods that are running on those instances, and any firewalls and routes created by the Container Engine service at the time of container cluster creation.

Developers Console

To delete a container cluster using the Developers Console:

  1. Visit the Developers Console.
  2. Select the appropriate project.
  3. In the left navigation menu, select Compute and then Container Engine. Your clusters are listed.
  4. Select the checkbox next to the cluster to be deleted.
  5. Click Delete. At the confirmation prompt, click Delete again.

Gcloud command line interface

To delete a container cluster with gcloud beta, use the clusters delete sub-command:

$ gcloud beta container clusters delete NAME --zone ZONE

Passing cluster credentials to kubectl

Once your cluster has been created, you'll use the kubectl command to create and manage resources on the cluster.

If your cluster was created through the Developers Console, or with gcloud from a different machine, you'll need to run an additional command to make your credentials available to kubectl. Your default zone and cluster must be already set or should be passed as flags to the command.

$ gcloud beta container get-credentials
    [--zone ZONE] [--cluster CLUSTER_NAME]

You only need to run this once per cluster per machine; for example, if you created your cluster from your laptop, you'll need to run get-credentials on your desktop before you're able to access the cluster from that machine.
